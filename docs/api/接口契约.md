# API 接口契约

**项目**：人口分布热力与公共设施叠加分析系统  
**后端版本**：v1.5.0  
**交接日期**：2026-06-26（更新于 2026-07-01）  
**目标读者**：前端开发人员（Vue3 + Leaflet）

---

## 一、项目概述

本项目是一个**人口分布热力与公共设施叠加**的可视化分析系统。核心功能：

1. 人口密度热力图层（支持 GHSL / WorldPop 两套数据集切换）
2. 公共设施点位图层叠加（学校、医院、公园）
3. 设施服务区缓冲覆盖可视化
4. 供给盲区识别与高亮
5. 供需统计指标展示（覆盖率、人均设施量）

**前后端交互方式**：前端通过 HTTP GET 请求调用后端 RESTful API，后端返回 JSON 数据，前端负责渲染为地图图层和统计图表。

---

## 二、技术栈总览

| 层级 | 技术 | 说明 |
| --- | --- | --- |
| **前端** | Vue3 + Leaflet | 底图优先使用天地图 |
| **后端** | Python FastAPI + Uvicorn | REST API 服务 |
| **默认数据后端** | 本地 GeoJSON 文件 | shapely + numpy，无需数据库，开箱即用 |
| **可选数据后端** | PostgreSQL + PostGIS | 空间数据存储与查询（`DATA_BACKEND=postgis`） |
| **坐标系** | **WGS84 (EPSG:4326)** | 经纬度（lng, lat），贯穿全链路 |
| **数据源** | WorldPop 2020（人口）+ OSM（设施）| 已预处理为本地 GeoJSON |

## 四、通用参数约定

### 4.1 bbox（经纬度范围框）

所有涉及空间查询的接口均使用 `bbox` 参数限定查询范围。

```
格式：minLng,minLat,maxLng,maxLat
示例：116.3,39.8,116.5,40.0
```

> **注意**：坐标系为 WGS84，顺序是 **经度、纬度**（与 Leaflet `L.latLngBounds` 的 `toBBoxString()` 一致）。

### 4.2 facility_type（设施类型）

| 值 | 含义 | 默认缓冲半径 |
| --- | --- | --- |
| `school` | 学校 | 1000 米 |
| `hospital` | 医院 | 2000 米 |
| `park` | 公园 | 500 米 |

### 4.3 buffer_radius（缓冲半径，可选）

设施服务半径（单位：米）。不传则使用上表默认值。

### 4.4 dataset（数据集，可选）

| 值 | 含义 |
| --- | --- |
| `worldpop`（默认） | WorldPop 人口网格数据 |
| `ghsl` | 全球人类居住层（Global Human Settlement Layer） |

## 五、API 接口详解

### 5.1 人口密度热力数据

> 获取指定范围内的人口密度格网点，用于前端渲染热力图层。

```
GET /api/v1/population/heatmap
```

**请求参数**：

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| bbox | string | ✅ 是 | — | 范围框：`minLng,minLat,maxLng,maxLat` |
| zoom | integer | ✅ 是 | — | 地图缩放级别，范围 `1 ~ 18` |
| dataset | string | ❌ 否 | `worldpop` | 数据集：`worldpop` 或 `ghsl` |

**响应字段**：

| 字段路径 | 类型 | 说明 |
| --- | --- | --- |
| code | integer | `200` 表示成功 |
| data.dataset | string | 实际使用的数据集 |
| data.bbox | array[4] | `[minLng, minLat, maxLng, maxLat]` |
| data.resolution | integer | 建议渲染分辨率，由 `max(10, 1000/zoom)` 计算 |
| data.points | array | 人口密度点列表（最多 10000 条） |
| data.points[].lng | float | 经度 |
| data.points[].lat | float | 纬度 |
| data.points[].value | float | 人口密度（人/km²） |

**请求示例**：

```
GET /api/v1/population/heatmap?bbox=116.3,39.8,116.5,40.0&zoom=12&dataset=worldpop
```

**响应示例**：

```json
{
  "code": 200,
  "data": {
    "dataset": "worldpop",
    "bbox": [116.3, 39.8, 116.5, 40.0],
    "resolution": 83,
    "points": [
      { "lng": 116.32, "lat": 39.85, "value": 4200.5 },
      { "lng": 116.35, "lat": 39.88, "value": 3100.0 }
    ]
  }
}
```

> **前端提示**：`points` 数组可直接用于 Leaflet.heat 插件或转为 Canvas 热力渲染。`resolution` 可用来控制点密度——zoom 越大范围越小，resolution 越小点越密。

### 5.1b 人口密度点查询（F12）

> 查询指定坐标所属 100m 网格的人口密度值，基于 STRtree 空间索引。

```
GET /api/v1/population/at-point
```

**请求参数**：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `lng` | float | ✅ | 经度（114.0–114.8） |
| `lat` | float | ✅ | 纬度（30.3–30.8） |

**响应字段**：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `data.lng` | float | 查询经度 |
| `data.lat` | float | 查询纬度 |
| `data.pop_density` | float | 人口密度（人/km²），无数据时为 0 |
| `data.cell_lng` | float | 所在网格中心经度 |
| `data.cell_lat` | float | 所在网格中心纬度 |
| `data.note` | string | 无数据时的说明文字 |

**示例**：

```
GET /api/v1/population/at-point?lng=114.40&lat=30.50
```

```json
{
  "code": 200,
  "data": {
    "lng": 114.4,
    "lat": 30.5,
    "pop_density": 2350.5,
    "cell_lng": 114.40072,
    "cell_lat": 30.50426
  }
}
```

### 5.2 公共设施列表

> 分页查询 bbox 范围内的公共设施点位，用于前端叠加设施标记图层。

```
GET /api/v1/facilities
```

**请求参数**：

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| bbox | string | ✅ 是 | — | 范围框 |
| facility_type | string | ❌ 否 | — | `school` / `hospital` / `park`；不传返回全部 |
| page | integer | ❌ 否 | `1` | 页码，从 1 开始 |
| page_size | integer | ❌ 否 | `500` | 每页条数，范围 `1 ~ 2000` |

**响应字段**：

| 字段路径 | 类型 | 说明 |
| --- | --- | --- |
| code | integer | `200` 表示成功 |
| data.total | integer | 符合条件的设施总数 |
| data.page | integer | 当前页码 |
| data.page_size | integer | 每页条数 |
| data.facilities | array | 设施列表 |
| data.facilities[].id | string | 设施唯一 ID（如 `osm_node_123456`） |
| data.facilities[].name | string | 设施名称 |
| data.facilities[].type | string | `school` / `hospital` / `park` |
| data.facilities[].subtype | string | OSM 原始细分类型（如 `kindergarten`、`clinic`） |
| data.facilities[].lng | float | 经度 |
| data.facilities[].lat | float | 纬度 |
| data.facilities[].source | string | 数据来源文件 |

**请求示例**：

```
GET /api/v1/facilities?bbox=116.3,39.8,116.5,40.0&facility_type=school&page=1&page_size=100
```

**响应示例**：

```json
{
  "code": 200,
  "data": {
    "total": 42,
    "page": 1,
    "page_size": 100,
    "facilities": [
      {
        "id": "osm_node_123456",
        "name": "北京第二实验小学",
        "type": "school",
        "subtype": "school",
        "lng": 116.38,
        "lat": 39.91,
        "source": "OSM/学校医院"
      }
    ]
  }
}
```

> **前端提示**：用户可能缩放/拖动地图，建议监听地图 `moveend` 事件，用新 bbox 重新请求。`total` 用于判断是否需要翻页。

### 5.3 供需统计

> 以设施服务半径为缓冲区，统计覆盖人口、覆盖率及千人设施量，用于仪表盘/统计面板展示。

```
GET /api/v1/analysis/supply-demand
```

**请求参数**：

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| bbox | string | ✅ 是 | — | 范围框 |
| facility_type | string | ✅ 是 | — | `school` / `hospital` / `park` |
| buffer_radius | integer | ❌ 否 | 按类型默认 | 服务半径（米） |

**响应字段**：

| 字段路径 | 类型 | 说明 |
| --- | --- | --- |
| code | integer | `200` 表示成功 |
| data.facility_type | string | 设施类型 |
| data.buffer_radius | integer | 实际缓冲半径（米） |
| data.total_population | integer | bbox 内总人口格网点数 |
| data.covered_population | integer | 被服务区覆盖的人口格网点数 |
| data.coverage_rate | float | 覆盖率 = covered/total（保留 3 位小数） |
| data.facility_count | integer | bbox 内该类型设施总数 |
| data.per_capita | float | 每千人设施量（保留 3 位小数） |
| data.per_capita_unit | string | 固定值：`"facility per 1000 persons"` |

**请求示例**：

```
GET /api/v1/analysis/supply-demand?bbox=116.3,39.8,116.5,40.0&facility_type=school
```

**响应示例**：

```json
{
  "code": 200,
  "data": {
    "facility_type": "school",
    "buffer_radius": 1000,
    "total_population": 15200,
    "covered_population": 11400,
    "coverage_rate": 0.75,
    "facility_count": 18,
    "per_capita": 1.184,
    "per_capita_unit": "facility per 1000 persons"
  }
}
```

> **前端提示**：`coverage_rate` 和 `per_capita` 适合用进度条/仪表盘展示。可以分别对 school/hospital/park 各请求一次做对比面板。

### 5.4 供给盲区识别

> 识别 bbox 内人口密度超过阈值但未被设施服务区覆盖的区域，返回 GeoJSON FeatureCollection。

```
GET /api/v1/analysis/blind-spots
```

**请求参数**：

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| bbox | string | ✅ 是 | — | 范围框 |
| facility_type | string | ✅ 是 | — | `school` / `hospital` / `park` |
| buffer_radius | integer | ❌ 否 | 按类型默认 | 服务半径（米） |
| pop_threshold | float | ❌ 否 | `1000.0` | 人口密度阈值（人/km²），高于此值才参与计算 |

**响应字段**：

| 字段路径 | 类型 | 说明 |
| --- | --- | --- |
| code | integer | `200` 表示成功 |
| data.facility_type | string | 设施类型 |
| data.pop_threshold | float | 使用的人口密度阈值 |
| data.blind_spot_count | integer | 识别出的盲区多边形数量 |
| data.blind_spots | object | **GeoJSON FeatureCollection** |
| data.blind_spots.type | string | `"FeatureCollection"` |
| data.blind_spots.features | array | GeoJSON Feature 数组 |
| data.blind_spots.features[].type | string | `"Feature"` |
| data.blind_spots.features[].geometry | object | **Polygon**（盲区凸包） |
| data.blind_spots.features[].properties.avg_population_density | float | 盲区内平均人口密度（人/km²） |
| data.blind_spots.features[].properties.area_km2 | float | 盲区估算面积（km²） |

**请求示例**：

```
GET /api/v1/analysis/blind-spots?bbox=116.3,39.8,116.5,40.0&facility_type=hospital&pop_threshold=2000
```

**响应示例**：

```json
{
  "code": 200,
  "data": {
    "facility_type": "hospital",
    "pop_threshold": 2000.0,
    "blind_spot_count": 1,
    "blind_spots": {
      "type": "FeatureCollection",
      "features": [
        {
          "type": "Feature",
          "geometry": {
            "type": "Polygon",
            "coordinates": [[[116.41, 39.82], [116.43, 39.82], [116.42, 39.85], [116.41, 39.82]]]
          },
          "properties": {
            "avg_population_density": 5300.2,
            "area_km2": 1.2
          }
        }
      ]
    }
  }
}
```

> **前端提示**：
> - `data.blind_spots` 是标准 GeoJSON，可直接用 `L.geoJSON(data.blind_spots)` 添加到 Leaflet 地图。
> - 盲区高亮建议使用红色/橙色半透明填充 + 加粗边框。
> - `properties` 中的密度和面积可绑定到 Popup/Tooltip 中展示。
> - **盲区聚类**：后端已实现基于网格的连通域聚类（`_cluster_blind_points`），可将散点拆分为多个独立盲区多边形，而非单一凸包。

### 5.5 设施缓冲覆盖区

> 将 bbox 内指定类型的所有设施缓冲区合并（ST_Union），返回完整服务覆盖几何体 GeoJSON。

```
GET /api/v1/analysis/coverage
```

**请求参数**：

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| bbox | string | ✅ 是 | — | 范围框 |
| facility_type | string | ✅ 是 | — | `school` / `hospital` / `park` |
| buffer_radius | integer | ❌ 否 | 按类型默认 | 服务半径（米） |

**响应字段**：

| 字段路径 | 类型 | 说明 |
| --- | --- | --- |
| code | integer | `200` 表示成功 |
| data.facility_type | string | 设施类型 |
| data.buffer_radius | integer | 实际缓冲半径（米） |
| data.coverage_geojson | object | 合并后的覆盖区 **GeoJSON Geometry** |
| data.coverage_geojson.type | string | 有设施时 `MultiPolygon`；无设施时 `GeometryCollection` |

**请求示例**：

```
GET /api/v1/analysis/coverage?bbox=116.3,39.8,116.5,40.0&facility_type=park
```

**响应示例**：

```json
{
  "code": 200,
  "data": {
    "facility_type": "park",
    "buffer_radius": 500,
    "coverage_geojson": {
      "type": "MultiPolygon",
      "coordinates": [
        [[[116.30, 39.80], [116.31, 39.80], [116.31, 39.81], [116.30, 39.80]]]
      ]
    }
  }
}
```

**空结果时的响应**：

```json
{
  "code": 200,
  "data": {
    "facility_type": "park",
    "buffer_radius": 500,
    "coverage_geojson": { "type": "GeometryCollection", "geometries": [] }
  }
}
```

> **前端提示**：
> - ⚠️ `coverage_geojson` 是 **Geometry 对象**，不是 FeatureCollection。添加到 Leaflet 时需要包装：`L.geoJSON({type: "Feature", geometry: data.coverage_geojson, properties: {}})`。
> - 也支持直接使用 `L.polygon()` 解析坐标数组。
> - 空结果是 `GeometryCollection`（`geometries: []`），前端需做判空处理。

### 5.6 全类型覆盖区 / 盲区 / 供需统计

> `all` 分析模式用于把学校、医院、公园三类设施按各自服务半径一起参与计算。三类半径均为可选参数；不传时回退默认值：学校 1000 米、医院 2000 米、公园 500 米。

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | `/api/v1/analysis/coverage-all` | 三类设施服务区合并后的统一覆盖区，返回 FeatureCollection |
| GET | `/api/v1/analysis/blind-spots-all` | 高人口密度且不被任一类设施覆盖的全类型盲区 |
| GET | `/api/v1/analysis/supply-demand-all` | 三类设施合并覆盖下的供需统计 |

**公共请求参数**：

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| bbox | string | ✅ 是 | — | 范围框 |
| school_radius | integer | ❌ 否 | 1000 | 学校服务半径（米） |
| hospital_radius | integer | ❌ 否 | 2000 | 医院服务半径（米） |
| park_radius | integer | ❌ 否 | 500 | 公园服务半径（米） |
| pop_threshold | float | ❌ 否 | 3000.0 | 仅 `blind-spots-all` 使用的人口密度阈值 |

**请求示例**：

```text
GET /api/v1/analysis/coverage-all?bbox=114.17,30.38,114.65,30.70&school_radius=3000&hospital_radius=3000&park_radius=3000
```

**响应说明**：

- all 接口响应 `data.radii` 会返回实际使用的三类半径。
- `coverage-all` 的 `data.coverage_geojson` 是标准 GeoJSON FeatureCollection。
- `blind-spots-all` 的 `data.blind_spots` 是标准 GeoJSON FeatureCollection。

### 5.7 人口密度点查询（F13）

> 点击地图任意位置，查询该坐标所属人口栅格网格的人口密度值。

```
GET /api/v1/population/at-point
```

**请求参数**：

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| lng | float | ✅ 是 | — | 经度，范围 `113.5 ~ 115.1` |
| lat | float | ✅ 是 | — | 纬度，范围 `29.9 ~ 31.4` |

**响应字段**：

| 字段路径 | 类型 | 说明 |
| --- | --- | --- |
| code | integer | `200` 表示成功 |
| data.lng | float | 查询经度 |
| data.lat | float | 查询纬度 |
| data.pop_density | float | 人口密度（人/km²），不在栅格范围内时为 `0` |
| data.cell_lng | float | 匹配网格中心的经度（仅命中时返回） |
| data.cell_lat | float | 匹配网格中心的纬度（仅命中时返回） |
| data.note | string | 未命中时的说明文字 |

**请求示例**：

```
GET /api/v1/population/at-point?lng=114.35&lat=30.52
```

**命中响应示例**：

```json
{
  "code": 200,
  "data": {
    "lng": 114.35,
    "lat": 30.52,
    "pop_density": 4200.5,
    "cell_lng": 114.35012,
    "cell_lat": 30.51987
  }
}
```

**未命中响应示例**：

```json
{
  "code": 200,
  "data": {
    "lng": 114.35,
    "lat": 30.52,
    "pop_density": 0,
    "note": "该点不在人口栅格覆盖范围内"
  }
}
```

> **前端提示**：接口通过 STRtree 空间索引实现 O(log n) 查询，延迟极低。地图 `click` 事件中可直接调用，用 Popup 展示密度值；未命中时优雅降级。坐标范围验证基于洪山区周边，超出范围返回 422。

## 六、错误码一览

| HTTP 状态码 | 描述 | 触发条件 |
| --- | --- | --- |
| 200 | OK | 请求成功 |
| 400 | Bad Request | `bbox` 格式错误（非 4 个数值）/ `facility_type` 不在允许范围 |
| 422 | Unprocessable Entity | 参数类型错误（如 `zoom` 传入非整数、`page_size` 超出 1~2000 范围） |
| 500 | Internal Server Error | 数据库连接失败或 SQL 执行异常 |

> **前端建议**：统一封装请求函数，对 400/422 做 toast 提示，500 做"服务异常，请稍后重试"。

## 七、数据表结构

> 以下是前端可能需要了解的数据库字段，用于理解数据来源和字段含义。

### population_grid（人口密度格网）

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | SERIAL PK | 自增主键 |
| geom | GEOMETRY(Point, 4326) | WGS84 坐标点 |
| value | FLOAT | 人口密度（人/km²） |
| dataset | VARCHAR(20) | `ghsl` 或 `worldpop` |

### facilities（公共设施）

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | VARCHAR(50) PK | OSM node id（如 `osm_node_123456`） |
| name | VARCHAR(255) | 设施名称 |
| type | VARCHAR(20) | `school` / `hospital` / `park` |
| geom | GEOMETRY(Point, 4326) | WGS84 坐标点 |

## 八、前端开发建议

### 8.1 地图初始化

- **底图**：优先使用天地图（Tianditu），WGS84 坐标系
- Leaflet 默认 EPSG:3857，需注意天地图瓦片的坐标系配置
- 所有 API 返回的坐标均为 **经度在前、纬度在后**（`[lng, lat]`），与 GeoJSON 标准一致

### 8.2 图层管理

建议使用 Leaflet 的 `L.LayerGroup` 或 `L.FeatureGroup` 分层管理：

| 图层 | 数据来源 | 渲染方式 |
| --- | --- | --- |
| 人口热力层 | `/population/heatmap` | Leaflet.heat 插件 或 Canvas |
| 设施标记层 | `/facilities` | L.marker 或 L.circleMarker（按类型分色） |
| 覆盖区层 | `/analysis/coverage` | L.geoJSON（半透明蓝色填充） |
| 盲区层 | `/analysis/blind-spots` | L.geoJSON（半透明红色填充 + 边框） |

### 8.3 交互流程建议

```
用户缩放/拖动地图
    │
    ├─→ 监听 moveend → 取 map.getBounds() 转 bbox
    │       │
    │       ├─→ /population/heatmap   ──→ 更新热力图层
    │       └─→ /facilities           ──→ 更新设施标记图层
    │
    ├─→ 用户选择设施类型 + 勾选"显示覆盖区"
    │       └─→ /analysis/coverage    ──→ 叠加覆盖区图层
    │
    ├─→ 用户点击"盲区分析"
    │       └─→ /analysis/blind-spots ──→ 叠加盲区高亮图层
    │
    └─→ 用户查看统计面板
            └─→ /analysis/supply-demand ──→ 更新统计指标
```

### 8.4 性能注意事项

- **热力点数据**：`/population/heatmap` 最多返回 **10000 条**，zoom 较大时点密集，前端渲染前可做抽样或聚合
- **设施分页**：默认 `page_size=500`，若视野内设施很多，注意翻页加载
- **GeoJSON 接口**：`/analysis/coverage` 返回的是合并后的单个大几何体，顶点可能较多，建议简化渲染样式

### 8.5 bbox 获取代码片段（Vue3 + Leaflet）

```javascript
// 从 Leaflet 地图获取 bbox 字符串
function getBboxString(map) {
  const bounds = map.getBounds();
  const sw = bounds.getSouthWest();
  const ne = bounds.getNorthEast();
  return `${sw.lng},${sw.lat},${ne.lng},${ne.lat}`;
}

// 调用示例
const bbox = getBboxString(mapInstance);
const res = await fetch(
  `http://localhost:8000/api/v1/population/heatmap?bbox=${bbox}&zoom=${mapInstance.getZoom()}`
);
const json = await res.json();
```
