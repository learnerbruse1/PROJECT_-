# 前端文件说明（逐文件）

> 面向需要**修改 / 扩展**前端的开发者。技术栈：Vue 3（`<script setup>`）+ Vite + Leaflet 1.9 + leaflet.heat。
> 后端接口契约见 [api/接口契约.md](api/接口契约.md)，运行方式见 [../README.md](../README.md)。

## 整体架构与数据流

```
main.js → App.vue
            ├─ MapContainer.vue        创建 Leaflet 地图，provide('map')
            │    ├─ HeatmapLayer.vue   ┐ 各图层 inject('map')，watch(store) 自行取数渲染
            │    ├─ FacilityLayer.vue  │
            │    ├─ CoverageLayer.vue  │   数据来自 api.js → 后端
            │    └─ BlindZoneLayer.vue ┘
            ├─ LayerControl.vue        改 store（图层开关、分析类型、半径、阈值）
            ├─ StatsPanel.vue          调 store.loadStats() 展示供需
            ├─ InfoPanel.vue           展示 store.selected / 数据说明
            └─ Legend.vue              纯展示

           stores/mapStore.js  ← 所有组件共享的唯一状态源（reactive 单例）
           api.js              ← 唯一的后端调用出口
           utils/geoUtils.js   ← 类型元数据 / 配色 / 坐标工具
```

**核心机制**：状态集中在 `mapStore`。用户操作面板 → 改 store → 各图层组件的 `watch` 命中 → 调 `api` 取数 → 操作自己的 Leaflet 图层。组件之间**不直接通信**，全部通过 store 解耦。地图实例由 `MapContainer` 用 `provide('map', ref)` 下发，子图层 `inject('map')` 获取。

## 状态与数据层

### src/stores/mapStore.js — 全局状态（reactive 单例）

整个应用唯一的状态源，`import { store } from '.../mapStore.js'` 后直接读写。

**状态字段**：
| 字段 | 类型 | 含义 |
| --- | --- | --- |
| `ready` | `null\|bool` | 数据自检结果（null=检测中） |
| `status` | `obj` | `/meta/status` 原始返回（含 counts/files） |
| `boundary` | `geojson` | 研究区边界 |
| `bbox` | `str` | 当前视野 `minLng,minLat,maxLng,maxLat` |
| `zoom` | `int` | 当前缩放级别 |
| `dataset` | `str` | 人口数据集（`worldpop`） |
| `layers` | `obj` | 6 个图层可见性：`heatmap/school/hospital/park/coverage/blindzone` |
| `analysisType` | `str` | 覆盖区/盲区/供需分析的单一设施类型 |
| `bufferRadius` | `null\|int` | 服务半径覆盖值（null=按类型默认） |
| `popThreshold` | `int` | 盲区人口密度阈值 |
| `selected` | `null\|obj` | 点选要素详情 `{title,kind,type,rows:[{k,v}]}` |
| `stats` | `obj` | 三类设施供需结果 `{school,hospital,park}` |
| `blindCount` | `int` | 最近一次盲区数量 |
| `loading` | `obj` | 各模块加载态 |
| `error` | `str` | 全局错误提示（5s 自动消失） |

**actions**：
| 方法 | 作用 |
| --- | --- |
| `init()` | 启动调用：拉 `/meta/status`（设 ready）+ `/meta/boundary` |
| `loadStats()` | 并发拉 3 类设施的供需统计填入 `stats` |
| `setView(bbox, zoom)` | 由 MapContainer 在 `moveend` 时更新视野 |
| `toggle(layer)` | 翻转某图层可见性 |
| `setAnalysisType(t)` | 设分析设施类型 |
| `select(info)` / `clearSelect()` | 设置/清空点选详情 |
| `setError(msg)` | 弹出错误提示 |
| `effectiveRadius()` | 取实际半径（`bufferRadius` 或类型默认） |

**扩展点**：加新图层 → 在 `layers` 加键，并新建对应图层组件 `watch` 它。

### src/api.js — 后端调用出口

封装所有 GET 接口，统一拼参、错误处理、解包 `data`。Base URL 取 `import.meta.env.VITE_API_BASE_URL`（默认 `/api/v1`，开发期经 Vite 代理）。

**`api` 对象方法**：
| 方法 | 对应接口 |
| --- | --- |
| `status()` | `/meta/status` |
| `boundary()` | `/meta/boundary` |
| `heatmap(bbox, zoom, dataset='worldpop')` | `/population/heatmap` |
| `facilities(bbox, facility_type, page=1, page_size=2000)` | `/facilities` |
| `supplyDemand(bbox, facility_type, buffer_radius)` | `/analysis/supply-demand` |
| `blindSpots(bbox, facility_type, buffer_radius, pop_threshold)` | `/analysis/blind-spots` |
| `coverage(bbox, facility_type, buffer_radius)` | `/analysis/coverage` |

均 `async`，返回响应里的 `data`；网络失败/非 2xx 抛 `Error`（消息可直接 toast）。**新增后端调用统一加在这里**，不要在组件里散写 `fetch`。

### src/utils/geoUtils.js — 元数据与工具

纯函数与常量，无副作用。
- 常量：`HONGSHAN_CENTER`（[lat,lng]）、`HONGSHAN_BBOX`、`DEFAULT_ZOOM`。
- `FACILITY_TYPES`：三类设施的 `{label,color,radius,emoji}`（**配色/默认半径的唯一来源**）。
- `FACILITY_ORDER`、`SUBTYPE_LABELS`（OSM 细分→中文）。
- `HEAT_GRADIENT`（leaflet.heat 渐变）、`HEAT_LEGEND`（图例色阶）。
- 函数：`bboxStringFromMap(map)`（Leaflet→bbox 字符串）、`toHeatPoints(points)`（后端点→`[lat,lng,intensity]`，按批次最大值做 sqrt 归一）、`fmt(n)`（千分位）、`pct(rate)`（百分比）。

## 地图与图层组件

> 图层组件模板均为隐藏占位 `<span style="display:none"/>`——它们只管理 Leaflet 图层，不渲染 DOM。统一套路：`inject('map')` → `watch(map, refresh, {immediate:true})` 首次加载 → `watch(相关 store 字段)` 变化重载 → 用 `token` 防竞态 → `onBeforeUnmount` 清理。

### src/components/MapContainer.vue — 地图主容器
- **职责**：创建 `L.map`，加载底图（天地图矢量/影像 + OSM/高德备用，`L.control.layers` 可切换），绘制研究区边界并 `fitBounds` 自动定位（F1）。
- **关键**：`provide('map', mapRef)` 下发地图；`<slot v-if="mapRef">` 确保子图层在地图就绪后才挂载。`moveend`（防抖 300ms）→ `store.setView`。天地图密钥取 `import.meta.env.VITE_TIANDITU_KEY`。
- **扩展点**：加底图在 `buildBaseLayers()`；改定位逻辑在 `drawBoundary()`。

### src/components/HeatmapLayer.vue — 人口热力（F2）
- 依赖 `store.layers.heatmap / bbox / dataset`；调 `api.heatmap`；用 `toHeatPoints` 转换后喂 `L.heatLayer`（`leaflet.heat`，渐变取 `HEAT_GRADIENT`）。
- 注意：必须 `import L from '../leafletSetup.js'` 再 `import 'leaflet.heat'`。

### src/components/FacilityLayer.vue — 设施点 + 点选（F3/F4/F5）
- 依赖 `store.bbox`（取数）与 `store.layers.{school,hospital,park}`（显隐）。
- 一次性拉全部设施（`api.facilities(bbox,null,1,2000)`），按类型分到三个 `L.layerGroup`，按可见性增删；不随单类型开关重新请求。
- 每个点 `circleMarker`（颜色取 `FACILITY_TYPES`）+ hover tooltip + click → `store.select(...)`（F5）。

### src/components/CoverageLayer.vue — 服务覆盖区（F6）
- 依赖 `store.layers.coverage / analysisType / bufferRadius / bbox`；调 `api.coverage`。
- 把返回的 Geometry 包成 `Feature` 用 `L.geoJSON` 渲染（半透明，颜色取 analysisType）；空结果（`GeometryCollection`）跳过。

### src/components/BlindZoneLayer.vue — 盲区高亮（F7）
- 依赖 `store.layers.blindzone / analysisType / bufferRadius / popThreshold / bbox`；调 `api.blindSpots`。
- `L.geoJSON` 渲染红色半透明面；写 `store.blindCount`；每个盲区 click → `store.select`（密度/面积/格网数）。

## 面板与展示组件

### src/components/LayerControl.vue — 控制面板（左）
改 store 的中枢：6 个图层勾选（`store.toggle`）、分析设施类型分段按钮（`store.analysisType`）、服务半径滑块（`store.bufferRadius`，可恢复默认）、盲区人口阈值输入（`store.popThreshold`）。改动后相关图层 `watch` 自动重载。

### src/components/StatsPanel.vue — 供需统计（右上，F8）
挂载即 `store.loadStats()`；`watch([bbox,bufferRadius,ready])` 防抖 500ms 自动刷新。三张卡片展示各类覆盖率（进度条/配色按高低）、覆盖/总格网、千人设施量；点卡片 = `setAnalysisType`。

### src/components/InfoPanel.vue — 属性/说明（右下，F5/F9）
有 `store.selected` 时展示其 `rows` 键值表（可关闭）；否则展示静态数据来源说明（F9）。

### src/components/Legend.vue — 图例（左下）
纯展示：人口密度色阶（`HEAT_LEGEND`，随 heatmap 开关显隐）+ 设施/覆盖/盲区图例。

### src/App.vue — 根布局与自检门控
`onMounted` 调 `store.init()`。顶栏标题 + 状态徽标（检测中/就绪+计数/未就绪）+ 全局 loading。`store.ready===false` 时渲染「数据未就绪」遮罩并列出缺失文件（F11）；否则渲染地图 + 四角浮动面板 + 错误 toast。

### src/main.js — 入口
顺序：`import './leafletSetup.js'`（**必须最先**，暴露全局 L 供 leaflet.heat）→ Leaflet/全局 CSS → `createApp(App).mount('#app')`。

### src/leafletSetup.js — Leaflet 全局垫片
`import L from 'leaflet'` 后 `window.L = L`，再 `export default L`。解决 leaflet.heat（老式 UMD 依赖全局 L）在 ESM 下无法注册 `L.heatLayer` 的问题。**任何会触发 leaflet.heat 的地方都应从这里取 L。**

## 常见开发任务速查

- **加一个新图层**：`mapStore.layers` 加键 → 新建 `XxxLayer.vue`（套用图层组件模板，`inject('map')` + `watch` + token 防竞态）→ 在 `App.vue` 的 `<MapContainer>` 内引入 → `LayerControl.vue` 加勾选 → `Legend.vue` 加图例。
- **改设施配色/默认半径**：只改 `utils/geoUtils.js` 的 `FACILITY_TYPES`。
- **接新后端接口**：在 `api.js` 加方法 → 组件/store 调用。
- **指向远程后端联调**：`frontend/.env` 设 `VITE_BACKEND_ORIGIN=http://host:8000`，重启 dev。
- **热力效果调参**：`HeatmapLayer.vue` 的 `L.heatLayer` 选项（radius/blur/max）+ `geoUtils.toHeatPoints` 归一方式。
