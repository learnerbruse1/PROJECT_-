# 后端文件说明（逐文件）

> 面向需要**修改 / 扩展 / 调用**后端代码的开发者。接口的请求/响应契约见 [api/接口契约.md](api/接口契约.md)，部署与总览见 [../README.md](../README.md)。
>
> 技术栈：Python 3.11+ / FastAPI / Uvicorn；默认**文件后端**（shapely），可选 PostGIS（asyncpg）。
> 关键环境变量：`DATA_BACKEND`（`geojson` 默认 / `postgis`）、`DATA_DIR`、`DATABASE_URL`、`CORS_ORIGINS`。

## 模块依赖关系

```
main.py
 ├─ routers/statistics.py ─┐
 ├─ routers/meta.py ───────┤
 └─ db/provider.py  ◀──────┘   （路由只认 provider，不直接依赖具体后端）
      ├─ DATA_BACKEND=geojson → db/geojson_store.py   （shapely，读 data/）
      └─ DATA_BACKEND=postgis → db/postgis.py         （asyncpg，连 PostGIS）
      （meta 接口的 data_status / get_boundary 恒走 geojson_store）
```

调用方向：**前端 → 路由 → provider → 具体后端 → 数据**。新增后端只需实现同名的 5 个查询函数并在 `provider.py` 接上，路由层零改动。

---

## backend/app/main.py — 应用入口

**职责**：创建 FastAPI 实例，挂载中间件与路由，管理生命周期。

- `lifespan(app)`：启动时若为文件后端则 `load_data()` 预载数据（提前暴露缺文件问题）；关闭时 `close_pool()` 释放资源。
- CORS 中间件：来源由 `CORS_ORIGINS`（逗号分隔，默认 `*`）控制，仅放行 `GET`。
- 路由挂载：`statistics.router` 与 `meta.router`，统一前缀 `/api/v1`。
- `GET /`：健康检查，返回 `{code, data:{service, backend, docs}}`。

**如何调用/运行**：`uvicorn app.main:app --reload --port 8000`（cwd 必须是 `backend/`，因为内部用 `app.xxx` 绝对导入）。

**扩展点**：新增一组接口 → 建 `routers/xxx.py` 并在此 `include_router`。

---

## backend/app/db/provider.py — 数据后端选择器

**职责**：依据 `DATA_BACKEND` 决定 5 个空间查询函数来自哪个实现，对上层透明。

**导出**（供路由/`main` import）：
| 名称 | 说明 |
| --- | --- |
| `query_heatmap / query_facilities / query_supply_demand / query_blind_spots / query_coverage` | 5 个核心查询，签名见下方两个后端 |
| `close_pool` | 释放资源（文件后端为空操作） |
| `load_data` | 预载数据（= geojson_store.load） |
| `data_status / get_boundary` | **恒来自 geojson_store**（元数据始终读本地文件） |
| `BACKEND_NAME` | 当前后端名（`'geojson'`/`'postgis'`），用于日志/响应 |

**如何调用**：`from app.db.provider import query_heatmap, ...`。**路由层一律从这里导入，不要直接 import 具体后端。**

**扩展点**：接入新后端时在此加 `elif _BACKEND == 'xxx': from app.db.xxx import (...)`。

## backend/app/db/geojson_store.py — 文件后端（默认）

**职责**：读取 `DATA_DIR`（默认项目根 `data/processed/`）下的本地 GeoJSON，在内存中用 shapely 完成全部空间计算。**无需数据库，开箱即用。**

**模块级常量**：
- `DATA_DIR`：数据目录（`DATA_DIR` 环境变量可覆盖）。
- `POP_FILE / SCHOOL_HOSPITAL_FILE / PARK_FILE / BOUNDARY_FILE`：四个数据文件名。
- `TYPE_MAP`：OSM 细分类型 → 三大类映射（`school/kindergarten/college/university→school`，`hospital/clinic/doctors→hospital`，`park→park`）。
- `_LNG0/_LAT0`：局部等距投影原点（洪山区中心），缓冲计算以米为单位的换算基准。

**生命周期函数**：
- `load() -> None`：幂等加载人口点 / 设施 / 边界进内存。`main.lifespan` 启动时调用。
- `close_pool() -> None`：空操作（与 PostGIS 接口对齐）。

**核心查询**（均为 `async`，签名与 PostGIS 后端一致）：
| 函数 | 入参 | 返回 |
| --- | --- | --- |
| `query_heatmap(bbox, dataset)` | `bbox:list[float]`, `dataset:str` | `list[{lng,lat,value}]`，>1万点按等步长**均匀抽样** |
| `query_population_at_point(lng, lat)` | `lng:float`,`lat:float` | `{lng,lat,pop_density,cell_lng,cell_lat}` 或 `None`（STRtree 点查询） |
| `query_facilities(bbox, fac_type, page, page_size)` | `fac_type:str\|None` | `(total:int, list[{id,name,type,subtype,lng,lat,source}])` |
| `query_supply_demand(bbox, fac_type, radius)` | `radius:int` | `{facility_count, total, covered}` |
| `query_blind_spots(bbox, fac_type, radius, pop_threshold)` | `pop_threshold:float` | `list[Feature]`（Polygon/MultiPolygon，凸包扣除覆盖区） |
| `query_coverage(bbox, fac_type, radius)` | | GeoJSON 几何（MultiPolygon 或空 GeometryCollection） |

**元数据函数**：
- `data_status() -> dict`：`{ready, data_dir, files:{...}, counts:{...}, backend}`，供 F11 自检。
- `get_boundary() -> dict|None`：研究区边界 GeoJSON。

**内部工具**（下划线前缀，不对外）：`_to_metric/_to_lnglat_xy`（经纬度↔米）、`_in_bbox`、`_facilities_in_bbox`、`_coverage_metric`（合并缓冲）、`_cluster_blind_points`（盲区网格连通域聚类）。

**扩展点**：
- 换数据文件 → 改文件名常量或设 `DATA_DIR`。
- 调缓冲精度 → `_coverage_metric` 里的 `buffer(radius, quad_segs=16)`。
- 调盲区聚类粒度 → `_cluster_blind_points(cell_deg=0.01)`。

## backend/app/db/postgis.py — PostGIS 后端（可选）

**职责**：用 PostgreSQL + PostGIS 实现同样的 5 个查询；`DATA_BACKEND=postgis` 时启用。

- `get_pool() -> asyncpg.Pool`：全局单例连接池（`asyncio.Lock` + double-checked locking，`min_size=2/max_size=10`）。连接串取 `DATABASE_URL`。
- `close_pool()`：关闭连接池。
- 5 个查询函数：签名与 geojson_store **完全一致**，内部用 `ST_MakeEnvelope` 做 bbox 预过滤命中 GIST 索引，再 `ST_Buffer(::geography)` / `ST_Union` / `ST_Within` / `ST_ConvexHull` 做精确运算。

**前置条件**：需已建表（`sql/schema.sql`）并导入数据；导入脚本需另行编写（见 HANDOVER）。

**扩展点**：改 SQL 即可；注意保持返回结构与文件后端一致，否则路由层会取不到字段。

## backend/app/routers/statistics.py — 九个分析接口

**职责**：人口热力、点查询、设施列表、供需、盲区、覆盖区共 9 个 GET 路由；参数校验后委托 `provider`。

**模块级**：
- `DEFAULT_RADIUS = {"school":1000, "hospital":2000, "park":500}`：各类型默认服务半径（米）。
- `VALID_TYPES`：允许的设施类型集合。
- `_parse_bbox(bbox:str) -> list[float]`：解析逗号分隔 bbox，非 4 个数值抛 `HTTP 400`。
- `_check_type(fac_type)`：类型非法抛 `HTTP 400`。

**路由**（前缀 `/api/v1`）：
| 路由 | 必填参数 | 可选 |
| --- | --- | --- |
| `GET /population/heatmap` | `bbox`,`zoom(1-18)` | `dataset='ghsl'` |
| `GET /population/at-point` | `lng`,`lat` | —（F12 点击地图查人口密度） |
| `GET /facilities` | `bbox` | `facility_type`,`page=1`,`page_size=500(≤2000)` |
| `GET /analysis/supply-demand` | `bbox`,`facility_type` | `buffer_radius` |
| `GET /analysis/blind-spots` | `bbox`,`facility_type` | `buffer_radius`,`pop_threshold=1000.0` |
| `GET /analysis/coverage` | `bbox`,`facility_type` | `buffer_radius` |

所有响应统一 `{code:200, data:{...}}`。`buffer_radius` 不传时取 `DEFAULT_RADIUS[type]`。`coverage-all`、`blind-spots-all`、`supply-demand-all` 额外支持 `school_radius / hospital_radius / park_radius` 三个可选参数，分别控制全类型分析中三类设施的实际服务半径；不传时回退各自默认半径。

**扩展点**：新增分析接口在此加 `@router.get(...)`，数据访问下沉到 provider/后端实现。

---

## backend/app/routers/meta.py — 元数据接口

**职责**：数据自检与边界，供前端启动与定位。

- `GET /meta/status`：返回 `data_status()` + `query_backend`（F11 数据自检）。
- `GET /meta/boundary`：返回 `{boundary: GeoJSON}`；边界缺失抛 `HTTP 404`。

数据始终来自本地文件（即便查询后端是 PostGIS）。

## 常见开发任务速查

- **本地起后端**：`cd backend && uvicorn app.main:app --reload --port 8000` → <http://localhost:8000/docs>。
- **切到 PostGIS**：设 `DATA_BACKEND=postgis` + `DATABASE_URL=...`，建表导数后启动。
- **加一个分析接口**：①`provider`/后端实现查询函数 → ②`statistics.py` 加路由 → ③（如需）`docs/api/接口契约.md` 补契约。
- **换研究区/数据**：替换 `data/` 下 GeoJSON，更新 `geojson_store.py` 文件名常量与投影原点 `_LNG0/_LAT0`。
