```
┌──── 前端 frontend/ (Vue3 + Leaflet) ───────────────────────────────┐
│  App.vue                                                             │
│  ├─ 左侧边栏 (280px, 手风琴布局)                                       │
│  │   ├─ 👁 图层控制 → LayerControl(mode="layers")       [可折叠]       │
│  │   ├─ 📊 分析配置 → LayerControl(mode="analysis")     [可折叠]       │
│  │   ├─ 📈 供需统计 → StatsPanel (折叠式)               [可折叠]       │
│  │   └─ 人口查询提示                                                   │
│  └─ 地图区 (flex:1)                                                    │
│      ├─ MapContainer ─ {Heatmap/Facility/Coverage/BlindZone}          │
│      ├─ InfoPanel (点选详情浮层，右上角)                                  │
│      └─ Legend (图例，左下角)                                            │
│  状态: stores/mapStore.js   调用: api.js (11 方法)   工具: utils/geoUtils│
└────────────────────────────┬──────────────────────────────────────────┘
                              │ HTTP GET /api/v1/*
┌─────────────────────────────▼───── 后端 backend/ (FastAPI) ──────────┐
│  routers/{statistics,meta}.py → db/provider.py                        │

                               │ 读取
                    data/processed/ 成果 GeoJSON（人口/设施/边界）
```

## 二、一次请求的完整链路（以「人口热力」为例）

1. 用户缩放地图 → `MapContainer` 的 `moveend`（防抖 500ms）→ `store.setView(bbox, zoom)`。
2. `HeatmapLayer` 的 `watch` 命中 `[store.layers.heatmap, store.bbox, store.zoom, store.dataset]` → 调 `api.heatmap(bbox, zoom, dataset)`。
3. `api.js` 发 `GET /api/v1/population/heatmap?...` → Vite 代理到 FastAPI。
4. `routers/statistics.py` 校验参数 → `provider.query_heatmap` → `geojson_store.query_heatmap`（numpy 向量化 bbox 过滤 + 均匀抽样）。
5. 返回 `{code,data:{points,...}}` → `api.js` 解包 `data` → `toHeatPoints` 转换 → `L.heatLayer.setLatLngs` 渲染。

**人口密度点查询（F12）链路**：用户点击地图 → `MapContainer` 的 click 事件 → `api.populationAtPoint(lng, lat)` → 后端 `query_population_at_point` → STRtree 点查询 → 返回 `{pop_density, cell_lng, cell_lat}` → Leaflet Popup 展示。

**记住三条边界**：组件只读写 `mapStore`；所有后端调用只走 `api.js`；路由只认 `provider`。改任何一层，另两层基本不受影响。
